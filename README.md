# 公交乘客投诉反馈平台

## 项目简介

全栈公交乘客投诉反馈平台，支持乘客提交投诉与建议、客服处理工单、管理员查看统计看板。覆盖从投诉提交、工单流转、进度跟踪到满意度评分的完整闭环流程。

## 技术栈

- **前端**：Vite + React 18 + TypeScript（默认端口 5175）
- **后端**：Express + TypeScript + better-sqlite3（默认端口 3203）
- **认证**：JWT + bcryptjs
- **启动方式**：concurrently 并行启动前后端
- **代理**：前端 `/api` 请求代理到后端 3203 端口

## 启动方式

```bash
# 安装所有依赖（根目录 + server + client）
npm run install:all

# 同时启动前后端
npm run dev
```

启动后访问 http://localhost:5175

## 功能说明

### 乘客端
- 注册与登录
- 提交投诉/建议（选择类型、描述、事发线路、站点、时间）
- 查看"我的工单"列表及处理进度（待受理 → 处理中 → 已回复）
- 工单详情页查看进度跟踪
- 工单关闭后对处理结果打满意度评分（1-5 星）

### 客服端
- 查看所有待处理工单列表
- 按投诉类型和线路筛选工单
- 填写处理意见，修改工单状态

### 管理员后台
- 投诉类型分布饼图
- 高频问题线路排行榜
- 月度投诉趋势折线图
- 满意度统计

## 注意事项

- 数据库文件位于 `server/db.sqlite`，首次启动自动建表并 seed 测试数据
- 可通过 `VITE_PORT`、`VITE_API_PORT`、`PORT` 覆盖默认端口
- 所有样式写在 `client/index.html` 的 `<style>` 标签中，未使用 CSS 文件导入或第三方 UI 库
- 如需重置数据库，删除 `server/db.sqlite` 后重启服务
